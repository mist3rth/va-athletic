Règle absolue : si la propriété peut changer la taille ou la position d'un autre élément dans le flux, elle déclenche un reflow complet. transform et opacity sont les seules exemptions navigateur.

02 — Layout animations
layout prop — cibler au maximumÀ doser
// Du moins au plus coûteux — choisir le plus précis <motion.div layout="position" /> // repositionnement seul <motion.div layout="size" /> // redimensionnement seul <motion.div layout="preserve-aspect" /> // ratio fixe <motion.div layout /> // tout — plus coûteux

Chaque mesure layout déclenche un reflow sur l'ensemble du sous-arbre DOM. Préférer layout="position" dès que la taille ne change pas.

Toujours isoler dans React.memo
// Un re-render parent ne doit pas retriggerer les mesures const AnimatedCard = React.memo(({ id, children }) => ( <motion.div layout="position" layoutId={id} transition={{ type: 'spring', stiffness: 300, damping: 30 }} > {children} </motion.div> ))
LayoutGroup — synchroniser plusieurs composants
import { LayoutGroup } from 'framer-motion' // Synchronise les mesures entre composants distincts // sans les faire communiquer via state <LayoutGroup id="tabs"> <TabBar /> <TabContent /> </LayoutGroup>
layoutId — shared element transitionPattern fort
// Card dans une liste <motion.div layoutId=`card-${id}` onClick={() => setSelected(id)} style={{ borderRadius: 12 }} /> // Modal — même layoutId = Framer calcule la transition FLIP <AnimatePresence> {selected === id && ( <motion.div layoutId=`card-${id}` key="modal" style={{ borderRadius: 20 }} exit={{ opacity: 0 }} /> )} </AnimatePresence>

Framer Motion calcule automatiquement la transition FLIP entre les deux positions. borderRadius est interpolé. Pas de calcul de coordonnées manuel.

03 — Variants
Déclarer les variants hors du composantEssentiel
Correct — objet stable
// En dehors du composant
const fadeUp = {
  hidden: { opacity: 0, y: 20 },
  visible: { opacity: 1, y: 0 }
}

function Card() {
  return (
    <motion.div variants={fadeUp} />
  )
}
Mauvais — recréé à chaque render
function Card() {
  // Nouvel objet à chaque render
  // Force Framer à recalculer
  const variants = {
    hidden: { opacity: 0 },
    visible: { opacity: 1 }
  }
  return (
    <motion.div variants={variants} />
  )
}

Si les variants dépendent de props/state, utiliser useMemo avec les dépendances exactes pour éviter les recréations inutiles.

Stagger — propagation automatique aux enfantsPattern clé
const container = { hidden: { opacity: 0 }, visible: { opacity: 1, transition: { staggerChildren: 0.08, // délai entre enfants delayChildren: 0.15, // délai avant le premier when: 'beforeChildren' // parent visible en premier } } } const item = { hidden: { opacity: 0, y: 16 }, visible: { opacity: 1, y: 0, transition: { ease: [0.25, 0.46, 0.45, 0.94] } } } // Les enfants héritent initial/animate — inutile de les répéter <motion.ul variants={container} initial="hidden" animate="visible"> {items.map(i => ( <motion.li key={i.id} variants={item} /> ))} </motion.ul>
useAnimationControls — contrôle impératifAvec précaution
const controls = useAnimationControls() // Déclencher depuis un événement async (API, WebSocket...) const handleSuccess = async () => { await controls.start({ scale: 1.05 }) controls.start({ scale: 1, transition: { delay: 0.1 } }) } // Toujours stopper au démontage — obligatoire useEffect(() => () => controls.stop(), []) <motion.div animate={controls} />

Préférer les variants déclaratifs. useAnimationControls uniquement si l'animation dépend d'événements asynchrones.

04 — Scroll animations
whileInView vs useInViewÀ connaître
whileInView — déclaratif, 90% des cas
<motion.section initial={{ opacity: 0, y: 30 }} whileInView={{ opacity: 1, y: 0 }} viewport={{ once: true, // anime une seule fois margin: '-80px', // déclenche avant le viewport amount: 0.3 // 30% de l'élément visible }} transition={{ duration: 0.5, ease: 'easeOut' }} />
useInView — impératif, logique conditionnelle
const ref = useRef(null) const isInView = useInView(ref, { once: true, amount: 0.3 }) // Déclencher de la logique (pas seulement une animation) useEffect(() => { if (isInView) { loadHeavyComponent() trackEvent('section_viewed') } }, [isInView]) <div ref={ref}> ... </div>
useScroll + useTransform — scroll-linkedPerformance sensible
const ref = useRef(null) const { scrollYProgress } = useScroll({ target: ref, offset: ['start end', 'end start'] }) // useTransform = dérivé direct, zéro re-render const y = useTransform(scrollYProgress, [0, 1], ['0%', '-15%']) const opacity = useTransform(scrollYProgress, [0, 0.5, 1], [1, 1, 0]) // TOUJOURS style (pas animate) pour les valeurs scroll-linked <div ref={ref}> <motion.div style={{ y, opacity }} /> </div>

animate déclenche une transition à chaque changement — sur le scroll c'est catastrophique. Utiliser uniquement style pour les valeurs useTransform.

useSpring — lisser les valeurs scrollDouceur
const { scrollYProgress } = useScroll() // Spring lisse les à-coups du scroll natif const smoothProgress = useSpring(scrollYProgress, { stiffness: 100, damping: 30, restDelta: 0.001 }) const scaleX = useTransform(smoothProgress, [0, 1], [0, 1]) // Barre de progression de lecture — exemple classique <motion.div style={{ scaleX, transformOrigin: 'left', height: '3px' }} />
05 — Cleanup & bundle
AnimatePresence — règles strictesCritique
Correct — key présente
<AnimatePresence mode="wait">
  {isOpen && (
    <motion.div
      key="modal"
      exit={{ opacity: 0, y: 20 }}
    />
  )}
</AnimatePresence>
Bug — sortie ignorée
<AnimatePresence>
  {isOpen && (
    <motion.div
      exit={{ opacity: 0 }}
    />
    {/* Pas de key
    = sortie non détectée */}
  )}
</AnimatePresence>
Choisir le bon mode
// wait — attend la sortie avant l'entrée (modals, pages) <AnimatePresence mode="wait"> ... </AnimatePresence> // sync — entrée et sortie simultanées (tabs) <AnimatePresence mode="sync"> ... </AnimatePresence> // popLayout — pousse le layout pendant la sortie (listes) <AnimatePresence mode="popLayout"> ... </AnimatePresence>
Nettoyer les animations impérativesObligatoire
// MotionValue — unsubscribe obligatoire const x = useMotionValue(0) useEffect(() => { const unsub = x.on('change', (v) => doSomething(v)) return unsub // retourner directement la fonction }, []) // AnimationControls — stopper à l'unmount const controls = useAnimationControls() useEffect(() => () => controls.stop(), [])
Réduire le bundle — LazyMotion~50 kb → ~18 kb
// Import standard — 50 kb gzip import { motion } from 'framer-motion' // LazyMotion — charge uniquement les features utilisées import { LazyMotion, domAnimation, m } from 'framer-motion' // Wrapper à la racine (layout.tsx en Next.js) <LazyMotion features={domAnimation} strict> {children} </LazyMotion> // Dans les composants : m au lieu de motion <m.div animate={{ opacity: 1 }} />

domAnimation inclut les animations standard sans les features 3D et drag avancé. domMax pour tout inclure. strict avertit si on utilise motion au lieu de m.

06 — Accessibilité & SSR
useReducedMotion — non négociableObligatoire
import { useReducedMotion } from 'framer-motion' function AnimatedSection({ children }) { const reduced = useReducedMotion() const variants = { hidden: { opacity: reduced ? 1 : 0, y: reduced ? 0 : 24 // supprimer le mouvement, garder la visibilité }, visible: { opacity: 1, y: 0 } } return ( <motion.div variants={variants} initial="hidden" whileInView="visible" transition={{ duration: reduced ? 0 : 0.5 }} viewport={{ once: true }} >{children}</motion.div> ) }

Conserver les transitions opacity légères même en mode réduit — elles aident la perception des changements d'état. Supprimer uniquement les mouvements y/x/scale/rotate.

07 — Patterns & architecture
Composant AnimateIn — wrapper réutilisablePattern fondation
// AnimateIn.tsx — à placer dans /components/ui/ type Dir = 'up' | 'down' | 'left' | 'right' | 'none' const offsets: Record<Dir, { x?: number; y?: number }> = { up: { y: 24 }, down: { y: -24 }, left: { x: 24 }, right: { x: -24 }, none: {} } export function AnimateIn({ children, delay = 0, direction = 'up', duration = 0.5, }: { children: React.ReactNode delay?: number direction?: Dir duration?: number }) { const reduced = useReducedMotion() const hidden = { opacity: reduced ? 1 : 0, ...(!reduced && offsets[direction]) } return ( <motion.div initial={hidden} whileInView={{ opacity: 1, x: 0, y: 0 }} viewport={{ once: true, margin: '-60px' }} transition={{ duration: reduced ? 0 : duration, delay: reduced ? 0 : delay, ease: [0.25, 0.46, 0.45, 0.94] }} > {children} </motion.div> ) } // Usage <AnimateIn direction="up" delay={0.1}> <h2>Titre de section</h2> </AnimateIn>

Checklist avant mise en productionAudit
  • Variants déclarés hors des composants (ou useMemo)
  • useReducedMotion sur toutes les animations
  • initial={false} sur les éléments SSR Next.js
  • AnimatePresence avec key sur chaque enfant conditionnel
  • useMotionValue + useTransform pour scroll/curseur (pas setState)
  • style prop (pas animate) pour les valeurs useTransform
  • layout prop ciblé — "position" ou "size", jamais layout seul sur des listes
  • LazyMotion + domAnimation si bundle size critique
  • AnimatePresence sur listes > 20 items → virtualiser (react-window)
  • layout + borderRadius → auditer avec DevTools Performance
  • will-change sur trop d'éléments → saturation GPU mobile
  • setState dans onPan / onDrag → remplacer par useMotionValue
  • Animer top / left / width / height → utiliser x / y / scale
  • Controls impératifs sans cleanup dans useEffect
