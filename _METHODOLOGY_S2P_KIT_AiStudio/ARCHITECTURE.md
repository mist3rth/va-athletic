# ARCHITECTURE : Kit Méthodologique S2P

Ce kit est structuré pour une exécution séquentielle dès l'importation d'un projet Google AI Studio.

## 📁 Organisation des fichiers du Kit

### 01. 🛡️ `MANIFESTO.md` (Règles d'Engagement)
- **Objectif :** Brider l'IA pour éviter qu'elle ne "vibe code" le design.
- **Contenu :** Directives strictes sur le respect du CSS original, des contenus et de la structure visuelle.

### 02. 🌐 `DEV_OPS_SETUP.md` (Environnements)
- **Objectif :** Automatiser le boilerplate infrastructure.
- **Contenu :** Générateur de `.env`, scripts de création de branches (dev, staging, main) et workflows GitHub Actions.

### 03. 🔍 `SKILL_ANALYZER.md` (L'Intelligence d'Audit)
- **Objectif :** Transformer l'agent en expert d'analyse de code généré.
- **Contenu :** Prompt d'audit profond qui identifie les patterns, les composants UI isolables et les zones de nettoyage prioritaires.

### 04. 🏗️ `REFACTORING_MASTER.md` (Le Processus)
- **Objectif :** Guider la migration du monolithe vers une structure propre.
- **Contenu :** Étapes de découpage : `App.tsx` -> `/layouts` -> `/sections` -> `/components` -> `/ui`.

### 05. 🔐 `SHIELD_PRO.md` (Sécurité & Debug)
- **Objectif :** Sécuriser et faciliter le développement.
- **Contenu :** Implémentation d'Error Boundaries, de logs personnalisés et de validation des types TS.

### 06. ✨ `POLISH_GRAIL.md` (La Touche Finale)
- **Objectif :** Apporter l'excellence visuelle et technique.
- **Contenu :** Guide GSAP, optimisation des assets (WebP), SEO sémantique et A11Y.

## 🔄 Workflow d'utilisation
1. **Ouvrir le ZIP** dans Antigravity.
2. **Exécuter `01. MANIFESTO.md`** pour caler les bases.
3. **Lancer `03. SKILL_ANALYZER.md`** pour obtenir le plan de travail.
4. **Suivre le plan** via les autres modules du kit.
